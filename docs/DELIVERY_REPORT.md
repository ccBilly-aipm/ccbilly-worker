## 一、你现在可以用它做什么

- **你可以** 打开一个属于自己的深空科技感工作台，深色/浅色两套皮肤随手切换，还能收起侧边栏。
- **你可以** 新建任务、在看板上拖着卡片改状态、点开任务改进度和勾子任务——每一步系统都自动记一条「动态」，不用你手动写日志。
- **你可以** 把相关任务归到一个「合集」里，合集会自动算出总进度，一眼看到某条线推进到哪了。
- **你可以** 一键生成当天日报：系统自动把你今天所有任务动态分成「完成 / 推进 / 受阻」填好，还帮你预填明日计划；改两笔、点定稿，再点「复制为 Markdown」就能直接粘进飞书，格式不乱。
- **你可以** 一键生成周报，自动统计本周完成数、各合集投入占比，配两张图，能导出成 .md。
- **你可以** 在工作台里直接管理你的 Claude Code Skill——扫出 `~/.claude/skills` 里所有 skill，在线看和改，改之前自动备份、绝不会丢你的 `allowed-tools` 这些配置，改完对 Claude Code 立刻生效。
- **你可以** 看自己的「个人技能树」雷达图，一键记录「今天学了什么」；也能浏览 Obsidian 笔记并点着双链跳转。
- **你可以** 把开源应用（如 Excalidraw 白板）接进「应用中心」，新标签打开或直接内嵌使用；碰上不让内嵌的网站会自动给你「新窗口打开」的按钮，不会白屏。
- **你可以** 在后台一键 Git 提交/同步（永远不会强推坏你数据）、改称呼和默认主题、把整个数据打包成 zip 备份。
- **你可以** 按 `Cmd/Ctrl+K` 全局搜索任务/日报/技能，或快速新建任务、生成日报、切主题。

**最关键的一点**：所有数据都是 `vault/` 目录里的纯文本 Markdown 文件。你用 Obsidian 直接改任何一篇，工作台几秒内自动跟着变；换电脑 `git pull` 就恢复全部数据；重要内容随手粘进飞书。数据永远是你的，不锁在任何数据库里。

## 二、怎么启动

第一次：

```bash
corepack enable      # 启用 pnpm（只需一次）
pnpm install         # 装依赖
pnpm seed            # （可选）生成一套演示数据
pnpm dev             # 打开 http://localhost:3000
```

日常：直接 `pnpm dev`。

> 如果装依赖时提示 better-sqlite3 要编译，运行一次 `pnpm rebuild better-sqlite3` 即可。

## 三、日常怎么用、怎么同步

1. **日常**：在工作台里管理任务、生成日报周报。所有改动实时写进 `vault/`。
2. **配合 Obsidian**：把 `vault/` 作为一个 Obsidian 库打开（或软链进你现有库），双向编辑互不冲突。
3. **配合飞书**：日报/周报的「复制为 Markdown」按钮就是给飞书用的，定稿后一键复制粘贴。
4. **多设备同步**：用后台的「Git 同步」面板，或命令行 `git pull` → `pnpm install` → `pnpm dev`。
5. **备份**：后台「数据导出」一键下载整个 vault 的 zip。

## 四、需要你做的两件事

1. **配置 Git 远端**（一次性）：我已经在本地建好 Git 仓库并提交了全部代码与数据，但你还没有配 GitHub 远端。配好后执行一次：
   ```bash
   git remote add origin <你的 GitHub 仓库地址>
   git push -u origin main
   ```
   之后就能用后台的「同步」按钮一键推送了。

2. **设置后台口令**（建议）：复制 `.env.example` 为 `.env.local`，把 `ADMIN_PASSCODE` 改成你自己的口令。不设也能用（本机单人场景），但设了更稳妥。

## 五、已知限制（都不影响日常使用）

- **Skill 的「新建 / 删除」暂未开放写入**：按当初和你对齐的「先只读+备份」策略，目前 Skill 管理开放了扫描/查看/在线编辑（编辑前自动备份），新建和删除按钮先做成禁用占位。等你确认编辑功能安全无误，我再用一个小迭代开放它们——底层的安全守卫和备份机制都已就绪。
- **视觉素材是占位图**：空状态插画和 Logo 目前是我用纯 CSS/SVG 画的玻璃拟态占位图。等你用 Codex + GPT Image 2 生成正式素材（提示词见需求文档 Part B），放进 `public/assets/` 替换即可，位置我都预留好了。
- **后台口令仅防误触**：它只是防止本机误点，**不是真正的安全防护**。如果你哪天把工作台部署到公网，务必另加真正的认证层。
- **Lighthouse 性能分**：桌面端 99 分（达标），移动端模拟慢速网络下 87 分——这是实验室对网页字体加载的严苛惩罚，真实使用流畅（首屏 0.9s、零布局抖动）。

# V1.1 · 安全加固说明（2026-07-12）

> 项目开源后，陌生人可能拿它对外部署。这一版我把安全从「攻击者视角」重审了一遍——每处都先写测试**证明漏洞真的存在**，再动手修。下面用「以前的风险是什么、现在怎么防住的」逐条说人话。技术细节在 `docs/SECURITY_AUDIT.md`。

## 一、堵住的安全洞（S1）

1. **知识库笔记里的恶意脚本**
   - 以前：如果一篇笔记（比如从别处拷来、或多设备同步进来的）里藏了段恶意脚本，你一打开它就会在你浏览器里偷偷运行——能借机调用工作台的接口去动你的文件。
   - 现在：所有笔记内容在显示前都会被自动「清洗」，脚本、恶意链接一律剥掉，正常的加粗、代码块、双链都不受影响。

2. **改 Skill 时被骗着写到别处**
   - 以前：Skill 管理号称「防止跳出目录」，但我实测发现一个真漏洞——通过一个指向外部的快捷方式（软链），可以骗工作台把文件写到你 skills 目录**外面**（比如覆盖你的其它配置）。
   - 现在：改成用文件的「真实位置」校验，就算路径还不存在也先解析真身，这条路彻底堵死。

3. **反向代理被当跳板去摸你的内网**
   - 以前：应用中心的反向代理只检查了网址开头，可以被设成 `127.0.0.1` 或云服务器的内部地址，让工作台**替攻击者**去访问你本机的模型服务、或云上的敏感接口。
   - 现在：代理前先解析目标真实 IP，本机/内网/云元数据地址一律拒绝（除非你在后台**手动打开**「允许内网」，而且云元数据地址永远拒绝）；跳转也逐跳重新检查。

4. **最要命的：写操作全裸奔**
   - 以前：只有后台有口令，但**新建任务、改 Skill、Git 推送、导出数据**这些写操作**完全没有鉴权**。你自己本机用没事；可一旦有人把它部署到局域网或公网，等于把你整台机器的 skills 目录和 Git 凭据开放给了所有人。
   - 现在：加了分层保护——**本机用照样免登录，体验一点没变**；但只要检测到是从非本机地址访问、又没开鉴权，所有写操作一律拒绝并提示你去配置。真要对外用，设一个 `AUTH_MODE=passcode` + 强口令就全锁上了。

5. **依赖里的已知漏洞**
   - 以前：项目依赖的库里有 **2 个严重 + 12 个高危**已知漏洞（其中就有能远程执行代码的、能绕过我上面那套鉴权的）。
   - 现在：全部升级到干净版本，安全扫描**归零**，而且每次改代码 CI 都会重新验证。另外给网页加了标准安全头。

## 二、更结实、更快（S2）

- **一次涌进几百个文件不卡**：你在 Obsidian 里 `git pull` 同步进来几百篇笔记，工作台会合并成一次批量更新，不会一个个卡半天。
- **坏文件不白屏**：空文件、格式错乱、超大文件都只会安静地进「待修复」列表，绝不让整个应用崩掉。
- **实测很快**：我造了一份 2935 条的大数据集压测，仪表盘、任务列表这些页面的响应都在**几十毫秒**内，远优于目标。

## 三、别人敢用、能贡献（S3）

- 每次推代码自动跑全部测试（**147 + 22 = 169 项**），仓库首页有绿色徽章证明健康。
- README 放了 4 张真机截图，配了安全说明、贡献指南、报 bug 模板。
- 默认称呼从「B哥」改成中性的「朋友」（你在后台设成自己想要的）；Docker 部署默认开口令保护。

## 四、你现在可以放心做的事

- **自己本机用**：跟以前完全一样，`pnpm dev` 打开就用，什么都不用配。
- **多设备 Git 同步**：照旧。
- **真要分享/对外部署**：先设 `AUTH_MODE=passcode` 和一个强口令（`.env.local` 里，或用 Docker），README「三种部署姿势」一节有详细指引。

一句话：**本机体验没变，但从今天起它敢开源、敢部署了。**

# V2.0 · 双角色版（面向 PM + 创作者）人话说明（2026-07-12）

> 这一版把工作台从「你一个人用的工具」升级成「一个真正的产品」——同一套内核，两副面孔：**产品经理**和**自媒体创作者**。而且它现在能被 AI（Claude Code）直接指挥干活。对你个人的意义：这本身就是你 AI PM 求职的作品集主案例——双角色产品设计 + 多 Agent 协作开发 + MCP 生态接入，三条故事线都齐了。

## 一、你现在多了什么

**先选个角色，工作台就变样。** 第一次打开会让你三步选角色：PM、创作者、还是双修（都要）。选完，左边菜单和首页仪表盘就按你的角色定制。随时能在后台切换，**切换只是换个显示方式，你的数据一个字都不会动**。

**如果你是 PM（或在练 PM）：**
- **需求池**：把想法记成需求卡，给它打 RICE 四个分（覆盖面/影响/信心/成本），系统自动算总分并排序——高价值的排前面。需求还能从「收集→需求池→已排期→已交付」四段走。
- **周期 + 燃尽图**：给某个合集设个起止日期，它就变成一个冲刺周期，自动画出燃尽图（还剩多少任务 vs 理想进度）。
- **路线图**：所有周期在时间轴上横向铺开，今天一条竖线高亮。
- **决策日志**：每个重要决定按「背景/选项/决定/理由/复盘时间」模板记下来，到复盘日会提醒你回来看当初对不对。
- **会议纪要转任务**：把纪要里 `- [ ] @谁 做什么` 那些行，一键全变成任务。
- **模板包**：PRD、竞品分析、用户访谈、复盘、会议纪要，一键复制模板。

**如果你做自媒体：**
- **选题库**：灵感随手记，标热度、记切入角度。
- **内容管道**：选题→草稿→待发→已发→复盘，拖着卡片走。
- **发布日历**：月历上按发布日期摆内容，拖一下就改期。
- **一稿多平台**：选好平台（公众号/小红书/抖音/B站/X），自动给你每个平台的适配清单。
- **数据复盘**：发完手动录入各平台阅读/点赞，自动出跨平台对比图。
- **情报源**：订阅任意 RSS/JSON 源（配合 wewe-rss 把竞品公众号变成源），看到好文章一键「存为选题」。**安全上，抓取只走白名单，绝不会被诱导去访问你的内网。**

## 二、工作台现在能被 AI 直接指挥（MCP）

跑 `pnpm mcp`，把它接进 Claude Code，你就能直接对 AI 说：
- 「把这周的选题按 RICE 排个序」
- 「新建一个任务：调研 wewe-rss 部署」
- 「生成今天的日报草稿」

AI 会真的去读写你的工作台。写操作在你开了口令保护时需要凭据，安全可控。接法见 `docs/MCP.md`。

## 三、更顺手的小改进

- 命令面板（Cmd+K）里输入 `!` 加一句话，回车直接扔进收件箱，稍后再分诊。
- 按 `?` 看快捷键表。
- 任务筛选组合可以命名保存成「视图」，一键切回。
- 周报升级成四步向导：自动汇总数据 → 记亮点 → 记问题 → 定下周计划。

## 四、你完全不用担心的事

- **旧数据**：你 V1 时期的所有任务、日报照常打开、照常能用——新功能全是可选字段，不动老文件。（想稳妥可先跑 `pnpm migrate`，它会先把整个 vault 打包备份再迁移。）
- **本机体验**：跟以前一样，`pnpm dev` 打开就用。
- **安全**：新增的情报源联网、MCP 写入都复用了 V1.1 那套防护，上线前还专门做了一轮独立审核（`docs/REVIEW_V2.md`），没有遗留问题。

一句话：**同一个工作台，现在既能当产品经理的兵器库，也能当自媒体的作战室，还能被 AI 当工具使唤。**
